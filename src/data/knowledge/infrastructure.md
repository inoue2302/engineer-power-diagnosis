# インフラ・DevOps トレンド 2025-2026

## コンテナ・オーケストレーション
- Kubernetes がエンタープライズ標準。EKS / GKE のマネージド運用
- Docker Compose でのローカル開発 + K8s でのプロダクション構成
- Istio / Linkerd によるサービスメッシュ
- ArgoCD / FluxCD による GitOps が CI/CD の主流パターン

## クラウド・サーバーレス
- AWS Lambda / Google Cloud Run / Azure Container Apps
- Vercel / Cloudflare Workers でのエッジコンピューティング
- Terraform / Pulumi による IaC が必須スキル
- マルチクラウド戦略よりクラウドネイティブな最適化が現実的

## CI/CD
- GitHub Actions が CI/CD のデファクト
- Dagger によるポータブル CI パイプライン
- Feature Flags（LaunchDarkly / Unleash）で段階的リリース
- Trunk-based development + short-lived branches

## SRE・運用
- SLO/SLI ベースの運用が標準化
- Incident Management（PagerDuty / Opsgenie）のプロセス整備
- Chaos Engineering（Litmus / Chaos Mesh）による耐障害性検証
- Platform Engineering チームの台頭。Internal Developer Platform 構築

## セキュリティ
- Supply Chain Security（SBOM / Sigstore）
- Container Security（Trivy / Snyk Container）
- Secret Management（Vault / AWS Secrets Manager）
